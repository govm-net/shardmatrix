# ShardMatrix 技术规格文档

## 项目定位

ShardMatrix 是一个高性能的单链区块链平台，专注于可扩展性和效率。项目采用迭代开发策略，第一阶段专注于实现稳定可靠的基础区块链功能。

## 核心技术参数

### 网络参数
- **区块间隔**: 3秒（固定）
- **区块大小**: 2MB（初期），后续可调整
- **验证者数量**: 21个
- **网络类型**: 单链（非分片）

### 共识机制
- **算法**: DPoS（委托权益证明）
- **验证者选择**: 基于质押权重排序
- **出块方式**: 验证者轮流出块
- **确认机制**: 即时确认（简化版）
- **最小质押**: 10,000 代币

### 性能目标
- **第一阶段**: 稳定运行，功能完整
- **后续优化**: 根据实际需求调整性能指标
- **交易处理**: 支持基础转账交易

## 数据模型

### 区块结构
```go
type Block struct {
    Header       *BlockHeader
    Transactions []Hash
}

type BlockHeader struct {
    Number       uint64
    Timestamp    int64
    PrevHash     Hash
    TxRoot       Hash
    StateRoot    Hash
    Validator    Address
    Signature    []byte
}
```

### 交易结构
```go
type Transaction struct {
    From      Address
    To        Address
    Amount    uint64
    Fee       uint64
    Nonce     uint64
    Data      []byte
    Signature []byte
}
```

## 开发约束

### 第一阶段限制
1. **功能范围**: 仅实现基础区块链功能
2. **性能目标**: 以稳定性为主，暂不追求极致性能
3. **复杂特性**: 暂不实现分片、高级共识等复杂功能
4. **智能合约**: 暂不支持，后续阶段考虑

### 设计原则
1. **简单优先**: 优先选择简单可靠的实现方案
2. **模块化**: 保持良好的模块边界，便于后续扩展
3. **可测试**: 所有核心功能必须有完整测试
4. **文档齐全**: 代码和设计文档同步更新
